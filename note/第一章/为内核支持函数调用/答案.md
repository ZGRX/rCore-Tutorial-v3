3.ra是一个保存函数返回地址的寄存器，如果嵌套调用，那么返回地址被覆盖，如果 `f` 调用 `g` 之前不保存自己的 `ra`，等 `g` 返回后，`f` 就不知道怎么回到 `main` 了。
结论：
函数里如果还要调用别的函数，就要先把自己的 `ra` 保存起来，通常保存到栈里。
4.

5. RISC-V 寄存器约定怎么理解？

你先记这几个就够：

|寄存器|用途|谁负责保存|
|---|---|---|
|`ra`|返回地址|需要时保存，常由当前函数保存到栈|
|`sp`|栈指针|必须保持正确|
|`a0~a7`|参数 / 返回值|调用者保存|
|`t0~t6`|临时变量|调用者保存|
|`s0~s11`|长期保存的变量|被调用者保存|

最重要的是这句话：

> `t` 开头的寄存器，别人可能随便改；`s` 开头的寄存器，别人用完必须恢复。

再简化：

```
a：传参数、返回值t：临时用，不保证不变s：稳定用，调用后仍应保持原值ra：保存返回地址sp：指向当前栈顶
```

---

整体理解：

**函数调用会覆盖寄存器，所以 RISC-V 规定了一套保存规则。这样不同函数、不同编译器生成的代码才能互相正确调用。**
