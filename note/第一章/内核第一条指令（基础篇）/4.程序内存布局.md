### 程序内存布局
![[Pasted image 20260531002416.png]]
这张图就是**典型进程/程序的内存布局**，从低地址到高地址大概是：

```
低地址.text     代码：机器指令.rodata   只读数据：字符串常量、只读常量.data     已初始化的全局/静态变量.bss      初始值为 0 的全局/静态变量heap      堆：动态分配，向高地址增长空洞stack     栈：函数调用、局部变量，向低地址增长高地址
```

最重要的是这几个：

|区域|放什么|
|---|---|
|`.text`|函数代码、汇编指令|
|`.rodata`|只读常量，比如 `"hello"`|
|`.data`|有初始值的全局变量，比如 `static X = 3`|
|`.bss`|初始为 0 的全局变量，比如 `static mut BUF = [0; 4096]`|
|`heap`|`Box`、`Vec`、动态申请的内存|
|`stack`|函数调用栈、局部变量、返回地址|

箭头表示：

```
heap  向高地址增长stack 向低地址增长
```

中间留空，是为了让堆和栈有扩展空间。

不过要注意：这图是**概念图**，不是所有系统都完全长这样。  
在 rCore 早期内核里，你更应该重点看：

```
.text -> .rodata -> .data -> .bss -> boot stack
```

因为一开始未必已经有完整的 heap，也还没有普通用户进程那种成熟布局。
